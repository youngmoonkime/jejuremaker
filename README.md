
# 웹사이트 구축 과업지시서 (요구사항 정의서)

## 1. 사업 개요
*   **프로젝트명**: 제주 리메이커 (Jeju Remaker) - AI 기반 업사이클링 디자인 플랫폼
*   **사업 목적**: 폐기물 정보를 기반으로 AI가 디자인 도면과 3D 모델을 생성하여, 사용자가 쉽고 재미있게 업사이클링에 참여할 수 있는 플랫폼 구축
*   **구축 목표**: 웹 3D 뷰어, 생성형 AI(이미지/모델링), 커뮤니티 기능이 통합된 반응형 웹 서비스 구현

## 2. 기술 환경 (Tech Stack)
*   **프론트엔드**: React (TypeScript), Vite, Tailwind CSS
*   **백엔드/데이터베이스**: Supabase (Auth, PostgreSQL, Edge Functions)
*   **AI 엔진**: Google Gemini 2.0 (멀티모달 분석/생성), Tripo AI (Image-to-3D)
*   **파일 스토리지**: Cloudflare R2 (대용량 3D 및 이미지 자산 관리)
*   **배포 환경**: Cloudflare Pages 또는 Vercel

## 3. 주요 기능 요구사항

### 3.1. AI 디자인 위자드 (Magic Wizard)
*   **재료 분석 (Material DNA Scan)**
    *   사용자가 업로드한 폐기물 사진을 AI(Gemini)가 분석하여 재질, 색상, 특성을 추출해야 함.
    *   분석 중 로딩 상태를 시각화(애니메이션)하여 제공할 것.
*   **디자인 컨셉 생성**
    *   분석된 재료 데이터를 바탕으로 4가지 디자인 스타일(모던, 오가닉 등) 제안.
    *   선택한 스타일에 따라 AI가 고품질 예상 렌더링 이미지 4종 생성.
*   **도면 및 가이드 생성**
    *   최종 선택된 디자인의 제작 도면(Blueprint) 자동 생성.
    *   제작 단계별(준비-가공-조립) 텍스트 가이드 및 팁 제공.
*   **3D 모델링 생성**
    *   2D 이미지를 3D 모델(.glb)로 변환하는 AI 파이프라인 연동.
    *   생성된 모델은 즉시 웹 뷰어에서 확인 가능해야 함.

### 3.2. 인터랙티브 3D 뷰어
*   웹 브라우저 상에서 별도 설치 없이 3D 모델(.glb/.gltf) 구동.
*   **필수 기능**: 회전(Rotate), 줌(Zoom), 와이어프레임(Wireframe) 모드 토글.
*   배경 색상 변경 및 전체 화면 보기 지원.

### 3.3. 커뮤니티 및 라이브러리
*   **공개 갤러리 (Discovery)**
    *   다른 사용자가 만든 프로젝트를 탐색할 수 있는 피드형 UI.
    *   '좋아요', '조회수' 기반의 트렌딩 정렬 기능.
*   **내 서재 (My Library)**
    *   사용자가 생성한 프로젝트의 영구 저장 및 관리.
    *   생성된 모든 자산(이미지, 3D 파일, 도면)의 다운로드 기능 제공.
*   **메이커 랭킹**
    *   활동량 및 인기도를 기반으로 한 'Top Maker' 순위 표시.

### 3.4. 사용자 및 시스템 관리
*   **회원 시스템**: 이메일 및 소셜 로그인(Google 등) 지원 (Supabase Auth 연동).
*   **토큰(크레딧) 시스템**:
    *   AI 생성 시 토큰 차감 로직 구현.
    *   매일/매월 자동 충전 또는 활동 기반 보상 로직.
*   **데이터 지속성**: 프로젝트 생성 중 이탈해도 데이터가 손실되지 않도록 DB 트랜잭션 및 스토리지 연동 철저.

## 4. 디자인가이드 (UI/UX)
*   **컨셉**: Apple 스타일의 미니멀리즘, 유려한 애니메이션, 글래스모피즘(Glassmorphism) 적극 활용.
*   **반응형 웹**: 데스크탑, 태블릿, 모바일 등 모든 디바이스에 최적화된 레이아웃.
*   **다크 모드**: 시스템 설정 또는 사용자 토글에 따른 다크/라이트 모드 완벽 지원.

## 5. 산출물 (Deliverables)
1.  **소스 코드 원본**: GitHub 리포지토리를 통한 전체 소스 코드 (.git 포함).
2.  **데이터베이스 스키마**: Supabase 테이블 명세서 및 타입 정의 파일.
3.  **운영 매뉴얼**: 로컬 구동 방법 및 환경 변수(.env) 설정 가이드.
4.  **배포 완료된 URL**: 실제 접속 가능한 웹 서비스 주소.

## 6. 유지보수 및 기타
*   코드 내 하드코딩을 지양하고, 환경 변수(Config)를 통한 API 키 관리.
*   AI API 연동 오류 시 사용자에게 적절한 예외 처리 메시지 제공.
